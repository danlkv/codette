How does zoom behave in this layout?
exp3 · explanation
Since there's no fixed-height container, zoom has no effect on the layout geometry. The browser zooms the document normally — you pinch to zoom, then pan to read, exactly like a regular web page. The sticky input stays at the bottom of the viewport as you scroll. The main tradeoff: scroll-to-bottom on new messages requires scrolling window, not a sub-element. On mobile Chrome, this feels the most "native" during zoom.
