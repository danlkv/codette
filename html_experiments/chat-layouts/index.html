Chat Layout Experiments

Testing mobile viewport stability — open each on phone and try: scroll, keyboard, pinch-zoom

1. CSS Grid + 100dvh
body { display:grid; grid-template-rows: auto 1fr auto; height:100dvh }
Grid natively handles the 3-row layout. dvh shrinks with keyboard.
scroll ✓ zoom reflows
2. CSS Grid + 100svh
body { display:grid; grid-template-rows: auto 1fr auto; height:100svh }
Same grid, but svh = smallest possible viewport height. Layout never changes size — keyboard overlaps content instead of resizing it. Most stable on zoom.
scroll ✓ zoom stable ✓ keyboard overlaps
3. position:fixed header + footer
header/footer { position:fixed } + body scrolls
Most reliable cross-browser. Fixed elements ignore viewport changes entirely. Uses ResizeObserver to track footer height for bottom padding. Body is the scroll container.
scroll ✓ zoom stable ✓ keyboard ✓
4. Flex column + 100svh
body { display:flex; flex-direction:column; height:100svh } + min-height:0
Flex equivalent of layout 2. Requires min-height:0 on the message area (the critical flex scroll trick). svh keeps it stable. Closest to current webclaude structure.
scroll ✓ zoom stable ✓
5. Flex + dvh + JS visualViewport (current approach)
--app-h = visualViewport.height → body { height: var(--app-h) }
JS tracks the visual viewport and updates a CSS variable. Precise keyboard handling. Shows a debug overlay with current vp height and scale. Reflows on zoom because scale changes trigger resize events.
scroll ✓ keyboard precise ✓ zoom reflows
