Testing fix for zoom-shrink bug on Chrome mobile

The problem: On Chrome mobile, pinch-zoom fires visualViewport.resize with a smaller height value. If the app uses --app-height = visualViewport.height unconditionally, the layout shrinks every time the user zooms in.
Approach 1 — JS height with scale guard
exp1a · baseline SHOWS BUG
Updates --app-height on all visualViewport.resize events including zoom. Zoom-in → layout shrinks.
exp1b · scale guard FIX
Skips --app-height update when visualViewport.scale > 1.05. Keyboard resize still works; zoom does not shrink layout.
Approach 2 — static CSS viewport units (no JS)
exp2a · 100svh NO JS
height: 100svh — "small" viewport height, the most conservative value. Never reacts to zoom or chrome show/hide.
exp2b · 100dvh NO JS
height: 100dvh — "dynamic" viewport height. Tracks browser chrome show/hide via CSS. Not affected by pinch-zoom.
Approach 3 — document scroll (different paradigm)
exp3 · doc scroll ALT
No fixed-height container. Document scrolls natively. Sticky header + sticky input. Zoom works like a normal webpage.
