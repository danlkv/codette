What's the keyboard behavior like with dvh vs svh?
exp2b · explanation
dvh updates when the browser's address bar shows/hides (which happens when you scroll). This keeps the layout perfectly sized but can cause a brief reflow. svh is conservative — uses the smallest possible viewport height so the layout never needs to grow. For keyboards: both shrink when the keyboard appears on iOS (since iOS keyboard reduces dvh), but on Android Chrome the soft keyboard may not affect dvh — you'd need the visualViewport API for that.
